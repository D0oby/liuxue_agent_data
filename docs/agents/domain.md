# Domain Docs

How the engineering skills should consume this repo's domain documentation when
exploring the codebase.

## Layout

Single-context repo.

## Before exploring, read these

- `CONTEXT.md` at the repo root, if it exists
- `CONTEXT-MAP.md` at the repo root, if it exists
- Relevant ADRs under `docs/adr/`, if they exist
- The nearest applicable `AGENTS.md`

If these files do not exist, proceed silently. The domain-modeling workflows can
create them later when terms or architectural decisions are actually resolved.
